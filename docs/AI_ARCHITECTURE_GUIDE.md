# Актуальный стандарт модульности

Главный актуальный стандарт архитектуры проекта описан в `docs/MODULAR_ARCHITECTURE.md`.

Этот файл остаётся полезным как справочник по сетевым паттернам, input prediction и visual RPC, но любые решения по структуре компонентов, ответственности, `TargetCursor`, здоровью, башням и `GameNetworkManager` должны сверяться с `MODULAR_ARCHITECTURE.md`.

Если правило из этого файла противоречит `MODULAR_ARCHITECTURE.md`, приоритет имеет `MODULAR_ARCHITECTURE.md`.

# ШАБЛОН-ИНСТРУКЦИЯ ДЛЯ ИИ: Архитектура Мультиплеера S&box

«Я разрабатываю мультиплеерную игру в движке S&box (на базе актуальной Scene System). Главный приоритет — максимальная плавность (Zero Input Lag) для клиента. При написании компонентов (Component) строго соблюдай следующие архитектурные правила:

## 1. Передвижение и Ввод (Client Authority / Prediction)

- Ввод с клавиатуры/мыши и применение сил (движение CharacterController, вращение камеры) должны происходить в методе `OnUpdate()`.
- Весь локальный ввод обязательно оборачивается в проверку `if ( !IsProxy )`.
- Мы доверяем клиенту его передвижение. NetworkObject движка сам позаботится о сглаживании и передаче координат другим игрокам.
- Мы не пишем кастомную интерполяцию для Transform.

## 2. Анимация (Local Visuals)

- Никогда не передаем параметры аниматора (например, is_walking, speed) по сети через [Sync]. Это засоряет трафик и вызывает дергания.
- Анимация (например, CitizenAnimationHelper) считается локально у всех клиентов в методе `OnUpdate()` на основе текущей Velocity и WorldRotation персонажа.

## 3. Стрельба, Способности и Эффекты (Паттерн "Скрыть Пинг")

- Когда игрок нажимает кнопку действия (выстрел, каст), мы моментально проигрываем визуальные эффекты (звук, партиклы) на его клиенте внутри блока `if ( !IsProxy )`.
- Сразу после этого мы вызываем метод с атрибутом `[Rpc.Broadcast]`, передавая координаты прицела.
- Внутри RPC-метода мы проверяем: `if ( IsProxy ) { ПроигратьЭффекты(); }`. Это гарантирует, что остальные игроки увидят выстрел, а сам стрелявший не услышит звук дважды из-за задержки сети.

## 4. Характеристики, Урон и Важные Данные (Server Authority)

- Здоровье, мана, инвентарь и состояние смерти синхронизируются через свойства с атрибутом `[Sync]`.
- Изменять эти данные имеет право только сервер (Host). Логика урона обрабатывается либо через методы с атрибутом `[Authority]`, либо с предварительной проверкой `if ( Networking.IsHost )`.
- Клиент может только читать [Sync] переменные (например, для обновления своего UI), но никогда не должен сам отнимать себе здоровье.

## Стек

- C#
- S&box Component API
- [Sync]
- [Rpc.Broadcast]
- IsProxy

## Важные замечания

- Устаревшие методы (Transform.Position) не использовать, применять актуальные (WorldPosition, WorldRotation).
»
