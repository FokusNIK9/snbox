## 1. Блокировка изменения `[Sync]` переменных Сервером (Owner Authority)
**@tag=RPC, Authority, Sync, Networking, State**
*Дата добавления: 6 мая 2026 г.*

### Суть проблемы
Когда Сервер (Host) пытается напрямую изменить `[Sync]` переменную компонента, которым **владеет Клиент (Owner)**, сетевой движок s&box молча блокирует это изменение. Значение переменной не обновляется на Клиенте.

### Как было (НЕПРАВИЛЬНО)
```csharp
public void AddCargo( int amount ) {
    CargoCount += amount; // ОШИБКА: Сервер не имеет прав менять [Sync] переменную Клиента!
}
```

### Решение (ПРАВИЛЬНО)
Легально менять переменную может **только сам владелец**. Сервер должен отправить RPC команду `[Rpc.Owner]`.
```csharp
public void AddCargo( int amount ) {
    if ( IsProxy ) {
        AddCargoRpc( amount );
        return;
    }
    CargoCount += amount;
}

[Rpc.Owner]
private void AddCargoRpc( int amount ) {
    CargoCount += amount;
}
```

**Главное правило:** Всегда оборачивайте изменение `[Sync]` свойства в проверку `if ( IsProxy )` с перенаправлением вызова в приватный метод `[Rpc.Owner]`.
